Programming the movement & Collision

In Unreal Engine 5 using C++, movement for a third-person character involves creating a class inheriting from ACharacter, handling input to translate player commands into character movements (like MoveForward() and MoveRight() functions), configuring movement parameters via the UCharacterMovementComponent, managing collision and physics interactions, integrating animations to reflect different movement states, and optionally using Blueprints for fine-tuning or event handling. The character's class implements functions that interpret player input, determining the direction and intensity of movement, which are then translated into changes in the character's position and orientation within the game world through Unreal Engine's robust movement and physics systems.

Blender 3D Modeling & Animations

Using Blender, I designed a 3D model and imported it into the engine, along with animations reflecting different movements like walking, running, and idle stances. To smoothly transition between these animations based on the character's speed, you employed a "Blendspace" in Blueprints. This Blendspace acts as a visual representation of various animations mapped against different speed values. As Classy's speed changes, the Blendspace dynamically blends between the appropriate animations, smoothly transitioning between movements, such as walking slowly, running at moderate speed, or idling when stationary. This integration ensures that Classy's animations match its movement speed, enhancing the overall visual and realism of the character within the game environment.
