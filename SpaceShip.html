SPACESHIP 1v1 GAME

Python is a powerful and widely used programming language in the industry. In addition, it is easy to learn, and Python is used worldwide to teach programming fundamentals. Throughout most of my projects, I like to put the variables in the first few lines as it is easier for me to manage them and change them then.

PYGAME


This is one of my first programming projects. I created a game using the Pygame library in Python. Libraries are collections of files that can be referred to in programming code. The project's most challenging part was learning this particular library's syntax, as I was unfamiliar with it. However, once I became proficient, I was able to create a game where two players could play against each other on the same computer in a 1v1 Spaceship game.

Bullets

The code manages movement for two spaceships, yellow and red, in a Pygame environment. Two functions, yellow_handle_movement and red_handle_movement, interpret keyboard inputs (keys_pressed) to control each spaceship's movement based on specific keys pressed ('A', 'D', 'W', 'S' for yellow; arrow keys for red). These functions adjust the spaceships' positions by altering their x and y coordinates within defined boundaries, ensuring smooth movement while considering velocity (VEL).

FPS & Shooting


The function handle_bullets iterates through both yellow and red bullets, updating their positions based on their respective velocities. If a yellow bullet collides with the red player (red.colliderect(bullet)), it triggers an event for the red player being hit (RED_HIT) and removes the yellow bullet. Similarly, if a red bullet collides with the yellow player (yellow.colliderect(bullet)), it triggers an event for the yellow player being hit (YELLOW_HIT) and removes the red bullet. Additionally, if a bullet goes beyond the screen width, it's removed from the corresponding bullets list to prevent unnecessary processing.

Game loop

This code establishes the game loop using Pygame. It maintains the game running (run = True) and controls the frame rate with clock.tick(FPS). It checks for user events, specifically for quitting the game (pygame.QUIT), and if detected, sets run to False, ending the loop and quitting the Pygame application using pygame.quit().

Player Movement For Bullets


The game loop oversees everything, including player movements and actions. When players press 'E' or spacebar (pygame.KEYDOWN), bullets are fired from their positions and added to their bullet lists (yellow_bullets or red_bullets). These bullets are rectangles used for collision detection and are part of ongoing game mechanics. Potential sound effects for firing bullets are indicated by commented-out lines.

Health Feature

This code manages player health during the game. Upon collision events (RED_HIT or YELLOW_HIT), the respective player's health decreases by one. If a player's health reaches zero or below, it declares the winner based on who has health remaining: "Yellow Wins!" if the red player's health drops to or below zero, or "Red Wins!" if the yellow player's health reaches the same condition. The game concludes by displaying the winner and stopping the loop.
